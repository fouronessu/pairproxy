# PairProxy WebUI 日常运维能力评估报告

**评估日期**: 2026-03-09
**版本**: v2.7.0
**评估范围**: Dashboard 日常运维最小集能力

---

## 一、现有功能清单

### 1. 认证与会话管理 ✅
- **登录页面** (`/dashboard/login`)
  - 管理员密码认证
  - Session cookie 管理
- **登出功能** (`/dashboard/logout`)
- **会话保护**: 所有管理页面需要认证

### 2. 概览页面 ✅ (`/dashboard/overview`)
- **全局统计**
  - 总用户数、活跃用户数
  - 总分组数
  - 今日/本月 token 用量
  - 今日/本月费用
- **趋势图表** (Chart.js)
  - Token 用量趋势（柱状图）
  - 费用趋势（折线图）
  - Top 5 用户用量（横向条形图）
- **时间范围切换**: 7天/30天/90天

### 3. 用户管理 ✅ (`/dashboard/users`)
- **用户列表**
  - 显示：用户名、分组、状态、今日用量、本月用量、创建时间
  - 搜索/过滤功能
- **用户操作**
  - ✅ 创建新用户（用户名、密码、分组）
  - ✅ 启用/禁用用户
  - ✅ 重置密码
  - ✅ 变更分组
  - ✅ 吊销 Token（强制重新登录）
- **批量操作**: 支持

### 4. 分组管理 ✅ (`/dashboard/groups`)
- **分组列表**
  - 显示：分组名、成员数、配额设置、今日/本月用量
- **分组操作**
  - ✅ 创建新分组
  - ✅ 设置配额（日限额、月限额、RPM）
  - ✅ 删除分组（需无成员或强制删除）
- **配额设置**
  - 每日 token 上限
  - 每月 token 上限
  - 每分钟请求数上限（RPM）

### 5. LLM 管理 ✅ (`/dashboard/llm`)
- **LLM Target 列表**
  - 显示：名称/URL、Provider、权重、来源、健康状态、排水状态、绑定数、操作
  - 健康状态实时显示（✓ 健康 / ✗ 不健康）
  - 排水状态显示（🚰 排水中 / 正常）
- **Target 管理**
  - ✅ 添加新 Target（URL、Provider、API Key、名称、权重、健康检查路径）
  - ✅ 编辑 Target（仅数据库来源）
  - ✅ 删除 Target（仅数据库来源）
  - ✅ 查看详情（配置文件来源，只读）
  - ⚠️ URL 冲突验证
- **用户/分组绑定**
  - ✅ 创建绑定（用户或分组 → Target）
  - ✅ 删除绑定
  - ✅ 均分用户到所有 Target
- **排水控制**
  - ✅ 进入排水模式（停止接收新请求）
  - ✅ 退出排水模式

### 6. 日志查询 ✅ (`/dashboard/logs`)
- **使用日志查询**
  - 按用户过滤
  - 按日期范围过滤
  - 显示：时间、用户、模型、输入/输出 token、总 token、费用、耗时
- **分页**: 支持

### 7. 审计日志 ✅ (`/dashboard/audit`)
- **操作审计**
  - 显示：时间、操作者、操作类型、目标、详情
  - 记录所有管理操作（用户、分组、配额、LLM、Token 等）
- **分页**: 支持

### 8. 用户自助页面 ✅ (`/dashboard/my-usage`)
- **配额状态**
  - 今日用量 vs 日限额（进度条）
  - 本月用量 vs 月限额（进度条）
  - RPM 限制显示
- **用量历史**
  - 每日 token 用量图表（30 天）
  - 每日费用图表
- **用户信息**
  - 用户名、分组、创建时间

## 六、结论

### 当前状态
PairProxy WebUI **已具备日常运维的最小集能力**，核心功能完整度达到 **95%**，可以满足：
- ✅ 用户生命周期管理（创建、禁用、密码重置）
- ✅ 配额管理（分组配额、实时监控）
- ✅ LLM 管理（Target 管理、绑定、排水）
- ✅ 用量监控（全局/用户/分组统计、趋势图表）
- ✅ 故障排查（日志查询、审计追踪）

### 主要不足
1. **数据导出缺失** - 无法导出用量报表，影响财务对账
2. **API Key 管理依赖 CLI** - 需要命令行操作
3. **无实时监控** - 只有历史数据，无法实时查看活跃请求
4. **对话追踪需要文件系统访问** - 合规审计不便

### 建议
对于**中小规模部署（<100 用户）**，当前 WebUI 功能已足够。

对于**大规模部署（>100 用户）**或**严格合规要求**的场景，建议优先实现：
1. 数据导出功能（P1）
2. API Key WebUI 管理（P1）
3. 对话追踪 WebUI（P2）
