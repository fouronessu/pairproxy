## 2. 核心功能实现

### 2.1 功能清单

| 功能 | 状态 | 说明 |
|------|------|------|
| Worker 用量水印追踪 | ✅ 完成 | 防止 primary 宕机时数据丢失 |
| 健康检查优化 | ✅ 完成 | 可配置超时/阈值/恢复延迟 |
| 路由表主动发现 | ✅ 完成 | CProxy 定期轮询获取路由更新 |
| 请求级重试 | ✅ 完成 | 非流式请求自动故障转移 |
| 请求体大小限制 | ✅ 完成 | 10MB 限制防止 OOM（P0修复）|

### 2.2 技术实现

**用量水印追踪**:
- Reporter 批量上报机制
- 本地 DB 持久化未同步记录
- 网络恢复后自动补报
- 新增 `usageReportFails`/`pendingRecords` 指标

**健康检查优化**:
- 可配置 `health_check_timeout`（默认 3s）
- 可配置 `passive_failure_threshold`（默认 3次）
- 可配置 `recovery_delay`（默认 60s）
- 主动检查 + 被动熔断双重保护

**路由表主动发现**:
- CProxy 定期轮询 `/cluster/routing-poll` 端点
- 支持 304 Not Modified 节省带宽
- 使用 `shared_secret` 鉴权
- 自动更新本地路由缓存

**请求级重试**:
- 非流式请求自动重试到未尝试节点
- 可配置 `max_retries`（默认 2次）
- 可配置 `retry_on_status`（默认 502/503/504）
- 对用户透明，提升可用性

## 7. 运维支持

### 7.1 OpenClaw 自动化运维

**运维手册**: `OPENCLAW_OPS_GUIDE.md`（733行）

**运维任务**（5个）:
1. **健康检查**: 每分钟检查系统状态
2. **LLM 上游检查**: 区分系统故障和 LLM 服务问题
3. **数据库维护**: 完整性检查、VACUUM、日志清理
4. **自动备份**: 每日备份，保留 7 天
5. **故障响应**: 自动诊断和恢复

**AI 提示词**:
- 每个任务都包含详细的诊断和恢复提示词
- 支持智能根因分析
- 提供具体的修复步骤

**监控指标**（7个）:
- `sproxy_uptime`: SProxy 运行时间
- `health_check_failures`: 健康检查失败次数
- `llm_upstream_latency`: LLM 上游延迟
- `database_size_mb`: 数据库大小
- `disk_usage_percent`: 磁盘使用率
- `active_requests`: 活跃请求数
- `dropped_usage_count`: 丢弃的用量记录

**告警规则**（4个）:
- SProxy 服务停止（critical）
- 错误率超过 5%（warning）
- 用量数据丢失（critical）
- 磁盘空间不足（critical）

### 7.2 运维最佳实践

**定期检查**:
- 每周审查自动化操作
- 每月测试恢复流程
- 每季度审查配置

**安全措施**:
- 使用环境变量存储敏感信息
- 定期轮换 API Key 和 JWT
- 限制 OpenClaw 运行用户权限
- 记录所有自动化操作
