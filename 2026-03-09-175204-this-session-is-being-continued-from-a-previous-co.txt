      2542 +- ✅ **零配置**：自动检测并转换，无需手动开关                
      2543 +- ✅ **双向转换**：Anthropic → OpenAI（请求）+ OpenAI →      
           +Anthropic（响应）                                            
      2544 +- ✅ **流式支持**：完整支持 SSE 流式响应                     
      2545 +- ✅ **智能处理**：System 消息、结构化内容、finish_reason

      2546 +- ✅ **优雅降级**：转换失败时自动回退到原始请求              
      2547 +                                                             
      2548 +#### 16.9.2 配置示例                                         
      2549 +                                                             
      2550 +在 `sproxy.yaml` 中添加 Ollama target：                      
      2551 +                                                             
      2552 +```yaml

      2554 +  targets:

      2562 +    - url: "http://localhost:11434"                          
      2563 +      api_key: "ollama"              # Ollama 不需要真实 API

      2571 +#### 16.9.3 工作原理                                         
      2572 +                                                             
      2573 +**自动检测条件**：                                           
      2574 +- 请求路径 = `/v1/messages`（Anthropic 格式）                
      2575 +- 目标 provider = `ollama` 或 `openai`                       
      2576 +

      2578 +                                                             
      2579 +1. **请求转换**（Anthropic → OpenAI）：                      
      2580 +   - 路径：`/v1/messages` → `/v1/chat/completions`           
      2581 +   - System 字段：移动到 messages 数组第一条                 
      2582 +   - Content 结构：提取 text 类型内容                        
      2583 +   - 流式请求：自动注入 `stream_options.include_usage: true` 
      2584 +                                                             
      2585 +2. **响应转换**（OpenAI → Anthropic）：                      
      2586 +   - 非流式：JSON 结构映射                                   
      2587 +   - 流式：SSE 事件实时转换                                  
      2588 +   - finish_reason 映射：                                    
      2589 +     - `stop` → `end_turn`                                   
      2590 +     - `length` → `max_tokens`                               
      2591 +     - `content_filter` → `stop_sequence`                    
      2592 +

      2660 +                                                             
      2661 +**问题：请求失败，日志显示 "conversion failed"**             
      2662 +                                                             
      2663 +**原因**：请求 body 格式异常或包含不支持的字段。             
      2664 +                                                             
      2665 +**解决**：                                                   
      2666 +1. 检查日志中的详细错误信息                                  
      2667 +2. 协议转换失败时会自动回退到原始请求，不影响服务            
      2668 +3. 如果持续失败，检查 Ollama 服务是否正常运行                
      2669 +                                                             
      2670 +**问题：响应内容为空或格式错误**                             
      2671 +                                                             
      2672 +**原因**：Ollama 返回的响应格式与标准 OpenAI API             
           +不完全一致。                                                 
      2673 +                                                             
      2674 +**解决**：                                                   
      2675 +1. 升级 Ollama 到最新版本                                    
      2676 +2. 检查 Ollama 日志：`ollama logs`                           
      2677 +3. 验证模型是否正确加载：`ollama list`                       
      2678 +                                                             
      2679 +**问题：Token 统计不准确**                                   
      2680 +                                                             
      2681 +**原因**：Ollama 可能不返回 usage 信息。                     
      2682 +                                                             
      2683 +**解决**：这是 Ollama 的限制，sproxy 会记录 `input_tokens=0, 
           + output_tokens=0`。如需准确统计，建议使用支持 usage 字段的   
           +OpenAI 兼容服务。                                            
      2684 +

      2686 +                                                             
      2687 +**性能影响**：

      2689 +- 流式响应实时转换，无缓冲延迟                               
      2690 +- 对吞吐量无明显影响                                         
      2691 +

      2693 +1. **仅支持文本内容**：图片、工具调用等结构化内容会被忽略    
      2694 +2. **System 消息限制**：仅支持单个 system 字段，多个 system
